# OCR 识别比赛视频成绩

用户会提供一个或多个包含比赛视频的目录路径。你需要完成以下步骤：

## Step 1: 提取帧
// turbo
运行命令提取每个视频的最佳帧图片（支持多目录，空格分隔）：
```
python ai_ocr_timer.py "<目录1>" ["<目录2>" ...]
```
工作目录: `D:\cube\video-by-face`

## Step 2: 识别成绩
用 `view_file` 工具依次查看 `D:\cube\video-by-face\jpg\need_ai\` 下的每张 .jpg 图片。
读取每张图片中 LED 大显上的数字，记录成绩。

**成绩格式（能看清时）**:
- `S.mmm` (如 `3.698`) — 10秒以内
- `SS.mmm` (如 `43.243`) — 10~59秒
- `M:SS.mmm` (如 `1:46.315`) — 超过1分钟

**识别失败时，用以下格式代替成绩**，直接写入文件名：
| 情况 | 写入值 |
|------|--------|
| 大显是灭的 / 计时器未停止 | `FAIL_灭屏` |
| 大显被遮挡（手、身体等） | `FAIL_遮挡` |
| 画面模糊、对焦失败 | `FAIL_模糊` |
| 画面中根本没有大显 | `FAIL_无大显` |
| 数字残缺、无法确认 | `FAIL_残缺` |
| 其他说不清楚的原因 | `FAIL_未知` |

## Step 3: 展示结果
以表格形式展示识别结果，让用户确认：

| 序号 | 文件名 | 成绩/原因 |
|------|--------|-----------|
| 1 | xxx.MP4 | 4.098 |
| 2 | xxx.MP4 | FAIL_遮挡 |

失败项在表格中用 ⚠ 醒目标出。

## Step 4: 等待确认
等用户确认后才执行下一步。
如果用户说需要修正某个成绩或原因，用修正后的值。

## Step 5: 重命名
// turbo
用户确认后，运行命令重命名视频文件：
```
python ai_ocr_timer.py --apply "<成绩1>, <成绩2>, ..."
```
工作目录: `D:\cube\video-by-face`

例如：
```
python ai_ocr_timer.py --apply "4.098, FAIL_遮挡, 3.455"
```

失败的视频会被重命名为如 `FAIL_遮挡.MP4`，方便后续人工处理。
